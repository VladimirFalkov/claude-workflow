# claude-workflow

Плагин для [Claude Code](https://claude.ai/code), который упаковывает проверенный workflow разработки: спринтовый TDD, автоматическое ревью кода, структурированная документация, консистентные коммиты.

## Что это такое

**claude-workflow** — набор skills и agents для Claude Code, организующий работу над проектом по спринтам с чёткими правилами: тест до реализации, ревью после каждого шага, документация синхронизируется с кодом.

Плагин работает в двух режимах:

- **Greenfield** — новый проект с нуля. Sprint-workflow применяется с первого спринта.
- **Brownfield** — существующий проект без структуры Claude Code. Сначала запускается bootstrap-спринт (анализ кода, создание `.project-state/`, docs, первичный backlog), затем обычный sprint-workflow.

## Ключевые принципы

1. **Единое место конфигурации** — `CLAUDE.md` проекта. В нём декларируется стек, плагин подгружает соответствующие slice-skills.
2. **Fallback на universal rules** — если для слоя стека нет специфичного skill, применяются `testing-rules` и `coding-rules`. Работа не прерывается.
3. **Ленивый рост** — новый slice-skill появляется только когда накопится 2+ реальных кейса в этом слое. Правила не выдумываются заранее.
4. **TDD строго** — каждый skill и agent покрывается тестами.
5. **Плагин собирается по своим правилам** — sprint-workflow применяется к самому репозиторию плагина.

## Состав

**Skills (7):**
- `sprint-workflow` — основной цикл разработки по спринтам
- `audit-cycle` — ревью кода с маркерами 🔴🟡🟢
- `task-tracking` — ведение DONE/BACKLOG/RELEASE по спринтам
- `docs-sync` — синхронизация документации при коммите
- `git-commits` — Conventional Commits с русскоязычными описаниями
- `testing-rules` — универсальные правила написания тестов
- `testing-django` — правила тестирования Django-приложений

**Agents (7):**
- `spec-pm` — сбор требований и написание спецификации
- `architect` — архитектурные решения и ADR
- `planner` — разбивка архитектуры на атомарные шаги
- `implementer` — реализация шагов по TDD
- `reviewer` — независимое code review
- `researcher` — исследование кодовой базы
- `test-runner` — запуск тестов и анализ ошибок

## Установка

В v0.1.0 плагин подключается через флаг `--plugin-dir`:

```bash
claude --plugin-dir /path/to/claude-workflow
```

Документация по установке через marketplace (`/plugin install`) появится в v1.0.0 — см. [docs/installation.md](docs/installation.md).

## Статус

**v0.1.0** — bootstrap. Плагин в активной разработке: переносятся существующие skills и agents, строится инфраструктура. Не рекомендуется для использования в продакшен-проектах до v1.0.0.
